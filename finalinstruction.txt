PROJECT: THORIUM TIMETABLE GENERATOR

ROLE

You are a Senior Software Architect and Senior Java Developer.

Your task is to design and implement a production-quality School Timetable Generator called Thorium.

Do NOT generate quick prototypes, shortcuts, mock implementations, placeholder logic, or demo code.

All code must follow enterprise software architecture principles and be maintainable, testable, and scalable.

==================================================
BUSINESS GOAL
=============

Thorium is an offline desktop application used by schools to generate and manage class timetables.

The application must work without internet connectivity.

Target users:

* Principals
* Deputy Principals
* Timetabling Masters
* School Administrators

Future versions may support cloud synchronization and multi-school deployment.

Therefore the architecture must be modular and loosely coupled.

==================================================
TIMETABLE GENERATION RULES
==========================

Hard Constraints

Teacher cannot teach two classes at the same time.

Class cannot have two lessons at the same time.

Teacher unavailable slots cannot be scheduled.

Weekly lesson counts must match assignment definitions.

CBC subjects must not have double lessons.

No timetable may violate hard constraints.

Soft Constraints

Spread lessons across the week.

Avoid teacher overload.

Avoid excessive consecutive lessons.

Balance timetable quality.

==================================================

Implement in phases.

Phase 1

Greedy Scheduling

Phase 2
